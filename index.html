comandos de git que siempre olvido

porque cada 2 días busco en google "como deshacer commit git"

uso diario ramas cuando la cago extras

los que uso todos los días

básicamente lo que hago 50 veces al día sin pensar:

git status
git add .
git commit -m "fix bug"
git push

cuando cambié varias cosas pero solo quiero commitear algunas:

git add -p

esto te pregunta archivo por archivo si quieres agregarlo. súper útil cuando tocaste 20 archivos pero solo quieres commitear 3.

ver el historial

la versión corta que realmente puedes leer:

git log --oneline

con las ramitas dibujadas (no sé como se llama técnicamente):

git log --oneline --graph --all

para ver qué cambió en cada commit:

git log -p
tip: podes hacer un alias para no escribir tanto. tipo git config --global alias.lg "log --oneline --graph" y después solo escribís git lg

todo lo de ramas

crear una rama nueva y cambiarme a ella:

git checkout -b nombre-rama

o si usas la versión nueva de git:

git switch -c nombre-rama

ver todas las ramas (incluidas las del repo remoto):

git branch -a

borrar una rama local cuando ya no la necesito:

git branch -d nombre-rama

si git se pone denso y no te deja borrarla porque "hay cambios sin mergear" o algo así:

git branch -D nombre-rama

borrar una rama del repo remoto (tipo github):

git push origin --delete nombre-rama

merge vs rebase

merge es como juntar dos ramas y hace un commit extra de "merge". es seguro y no cambia el historial:

git checkout main
git merge mi-feature

rebase es como agarrar tus commits y ponerlos arriba de la otra rama. hace que el historial quede más limpio pero puede ser quilombo si no sabés lo que estás haciendo:

git checkout mi-feature
git rebase main
ojo: nunca hagas rebase de commits que ya pusheaste y otras personas descargaron. vas a romper todo y se van a enojar.

cuando la cago (seguido)

me arrepentí del último commit

si querés deshacer el commit pero mantener los cambios:

git reset --soft HEAD~1

si querés tirar todo a la basura y volver como estaba antes:

git reset --hard HEAD~1

cambié cosas y ahora quiero volver todo para atrás

git reset --hard HEAD

ojo que esto borra TODO lo que cambiaste. no hay vuelta atrás después de esto.

agregué un archivo al staging y quiero sacarlo

git restore --staged archivo.js

necesito guardar cambios pero no commitear todavía

esto pasa cuando estás en el medio de algo y te piden que arregles otra cosa urgente:

git stash

después cuando terminés con lo urgente, recuperas tus cambios:

git stash pop

ver todos los stash que tenés guardados:

git stash list

borré algo importante y necesito recuperarlo

git guarda TODO. hasta lo que borraste. esto te salva la vida:

git reflog

ahí vas a ver toda la historia de lo que hiciste. buscas el commit que querés y:

git reset --hard abc123

(donde abc123 es el hash del commit)

en serio: git reflog me salvó como 100 veces. cuando pensás que cagaste todo mal, reflog es tu amigo.

quiero un commit de otra rama

cherry-pick es para eso. copias un commit específico a tu rama actual:

git cherry-pick abc123

cosas extras que uso a veces

ver diferencias

lo que cambié pero todavía no agregué:

git diff

lo que ya está en staging:

git diff --staged

diferencia entre dos commits:

git diff abc123 def456

buscar algo en el código

git grep "función"

esto busca en todos los archivos del proyecto. es re rápido.

configuración inicial

cuando configurás git por primera vez (o en una compu nueva):

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

para que git sea lindo con colores:

git config --global color.ui auto

tags (para versiones)

cuando sacas una versión nueva tipo v1.0:

git tag v1.0.0
git push origin v1.0.0

actualizar tu repo con lo del remoto

git pull

si te da conflictos y no querés hacer merge, probá con:

git pull --rebase

ver quién tocó cada línea de un archivo

git blame archivo.js

esto muestra quién modificó cada línea y cuándo. útil cuando algo está roto y querés saber quién lo rompió jaja.

dato: si trabajas con github o gitlab, configurá SSH keys. así no tenés que meter password cada vez que haces push.
